# 更新日志

## [1.0.0] - 2025-11-24

### 新增功能

#### 核心功能
- ✅ 多规则转发支持 - 支持创建、编辑、删除多个转发规则
- ✅ 灵活的 URL 匹配 - 使用正则表达式匹配 URL，支持捕获组替换
- ✅ XHR/Fetch 拦截 - 完整拦截网页的 XHR 和 Fetch 请求
- ✅ 请求转发 - 将拦截的请求转发到目标服务器
- ✅ Cookie 自动获取 - 自动从目标服务器获取 Cookie
- ✅ Cookie 手动配置 - 支持手动添加和管理 Cookie
- ✅ 自定义 Headers - 支持为转发请求添加自定义 HTTP Headers
- ✅ 请求/响应日志 - 记录所有拦截和转发的请求

#### 界面功能
- ✅ 多标签页界面 - 规则、日志、设置三个标签页
- ✅ 规则管理界面 - 创建、编辑、删除、启用/禁用规则
- ✅ Cookie 管理界面 - 查看、添加、删除 Cookie
- ✅ 日志查看界面 - 搜索、过滤、查看详细信息
- ✅ 导入导出功能 - 支持备份和导入规则配置

#### 数据管理
- ✅ 本地存储 - 所有数据保存到 Chrome 本地存储
- ✅ 自动持久化 - 规则和配置自动保存
- ✅ 日志管理 - 支持清空日志和限制日志数量

### 技术实现

#### 架构
- Service Worker 后台脚本处理请求转发
- Content Script 在页面和后台之间传递消息
- Injected Script 在页面上下文中拦截 fetch 和 XHR
- Popup UI 提供用户界面

#### 安全性
- 不保存明文 Cookie（仅在内存中处理）
- 所有数据本地存储，不上传到服务器
- 限制插件权限到最小必要范围

### 已知限制

- Icon 文件使用占位符，需要替换为实际 PNG 图片
- 某些特殊网页（如 Chrome 内置页面）无法使用插件
- 跨域请求可能受到浏览器 CORS 限制

## 开发者注意事项

### 代码风格
- 使用 ES6+ 语法（但不使用 ES6 模块在 Service Worker 中）
- 函数和类都有 JSDoc 注释
- 中文注释和英文代码混合

### 测试方式
1. 在 Chrome 中加载未封装的扩展
2. 打开浏览器开发者工具检查日志
3. 在 popup 中创建测试规则
4. 在网页控制台执行 fetch 或 XHR 请求

### 常见修改

#### 添加新管理器
1. 在 `managers/` 目录创建新文件
2. 创建类并实现相应方法
3. 在 `background.js` 中初始化类
4. 在消息处理器中添加对应的消息类型

#### 修改 UI
1. 编辑 `popup/popup.html` 添加新元素
2. 在 `popup/popup.css` 中添加样式
3. 在 `popup/popup.js` 中添加事件处理

#### 修改请求转发逻辑
1. 在 `background.js` 的 `handleInterceptRequest` 函数中修改
2. 在 `injected.js` 中修改拦截逻辑
